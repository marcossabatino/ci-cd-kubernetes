Como o Site é Observável

Este site não é apenas sobre observabilidade — é observável. Enquanto você lê, sua requisição está sendo:

1. Request chega no Nginx

Access log registrado em JSON

{ timestamp, status, latency, path, bytes }

2. Loki agrega logs

Logs buscáveis no Grafana

{ job="nginx", status="200" }

3. Prometheus scrapa métricas

Nginx exporter expõe métricas

http_requests_total, request_duration_seconds

4. Grafana visualiza

Dashboard mostra em tempo real

Requests/sec, Latency p99, Error rate

Fluxo de Deploy

  1. Você faz push para GitHub

    Commit na branch main triggers GitHub Actions

  2. CI roda testes/lint

    Valida HTML, otimiza assets

  3. Docker build

    Cria imagem Nginx com site inside

  4. Push para Docker Registry

    Imagem fica disponível para download

  5. Kubernetes aplica nova imagem

    Rolling update: pod antigo → novo, sem downtime

  6. Health checks validam

    Readiness probe: novo pod pronto? Se não, rollback

  7. Tráfego passa para novo pod

    Load balancer redireciona requisições

  8. Métricas atualizam em tempo real

    Grafana mostra o deploy em ação
