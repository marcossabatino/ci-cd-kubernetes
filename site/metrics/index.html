📈 Métricas: Medindo a Saúde

Números que indicam como seu sistema está se comportando

O que é uma Métrica?

Uma métrica é uma medição quantitativa de um aspecto do seu sistema em um ponto do tempo. Diferente de logs (eventos discretos), métricas são números contínuos que você coleta, armazena e analisa.

Exemplos:

  • http_requests_total — Quantas requisições processadas?
  • request_duration_seconds — Quanto tempo leva?
  • cpu_usage_percent — Quanto de CPU está usando?
  • memory_bytes — Quanto de memória?
  • database_connections_active — Quantas conexões DB abertas?

Logs vs Métricas vs Traces

Aspecto Logs Métricas Traces
O quê? Eventos textuais Números quantitativos Fluxo de requisições
Quando? Quando algo acontece Continuamente (a cada 15s) Início ao fim de request
Volume Alto (pode ser muito) Menor (agregado) Médio (por requisição)
Custo Caro (storage) Barato (muito agregado) Médio (amostragem)
Usar para? Debugging específico Trends e alertas Performance profiling

Tipos de Métricas (Prometheus)

Counter ⬆️

Número que só aumenta (nunca diminui)

http_requests_total 1,234,567
errors_total 234

Perguntas que responde:

  • "Quantas requisições no total?"
  • "Quantos erros ocorreram?"
  • "Qual a taxa de erros?" (com rate())

Gauge 📊

Número que sobe e desce (estado atual)

memory_bytes_used 1,073,741,824
cpu_percent 45.2
active_connections 234

Perguntas que responde:

  • "Quanto de memória está sendo usado?"
  • "Qual é a temperatura da CPU?"
  • "Quantas conexões ativas?"

Histogram 📈

Distribuição de valores (para percentis)

request_duration_seconds_bucket{le="0.1"} 100
request_duration_seconds_bucket{le="0.5"} 250
request_duration_seconds_bucket{le="1"} 300

Perguntas que responde:

  • "Qual é o p95 de latência?"
  • "Que % de requisições é mais rápido que 100ms?"

Summary 📌

Similar a histogram, mas com cálculos no cliente

request_duration_seconds_sum 1234.56
request_duration_seconds_count 5000
request_duration_seconds{quantile="0.5"} 0.23

Quando usar:

  • Quando precisa de quantis exatos
  • Menos agregável que histogram

PromQL — Queryando Métricas

PromQL é a linguagem para buscar e manipular métricas em Prometheus. Aqui estão as queries mais úteis:

Taxa de requisições por segundo

rate(http_requests_total[5m])

Quanto de tráfego está chegando?

P95 de latência

histogram_quantile(0.95, request_duration_seconds)

95% das requisições são mais rápidas que isto

Taxa de erro

rate(http_requests_total{status=~'5..'}[5m]) / rate(http_requests_total[5m])

Qual % de requisições falha?

CPU disponível

(100 - node_cpu_seconds_total{mode="idle"}) / 100

Qual % de CPU está sendo usado?

Memória livre

node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes

Que % de memória ainda está disponível?

Comparar 2 versões

rate(http_requests_total{version="1.0"}[5m]) vs rate(http_requests_total{version="2.0"}[5m])

A v2.0 está processando mais tráfego?

Melhores Práticas

1️⃣ Nomeie Bem

Nomes de métricas devem ser descritivos e seguir convenção:

{job}_{subsystem}_{unit}

Ex: http_requests_total, database_query_duration_seconds

2️⃣ Use Labels com Moderação

Labels permitem filtrar. Mas cuidado: muitos labels = explosão cardinálica.

http_requests_total{method="POST", status="200", endpoint="/api/users"}

❌ Evite: user_id (infinitas combinações)

3️⃣ Escolha Intervalo de Scrape

Prometheus coleta métricas a cada X segundos (default: 15s). Mais frequente = mais storage, mas mais precisão.

global: { scrape_interval: 15s }

4️⃣ Implemente Alertas

Métricas só são úteis se gerar ação. Configure alertas:

ALERT HighErrorRate IF rate(errors_total[5m]) > 0.05

Exemplo Prático: Node.js com prom-client

import client from 'prom-client';

// Criar métricas
const httpRequestCounter = new client.Counter({
  name: 'http_requests_total',
  help: 'Total HTTP requests',
  labelNames: ['method', 'status'],
});

const requestDuration = new client.Histogram({
  name: 'http_request_duration_seconds',
  help: 'HTTP request latency',
  buckets: [0.01, 0.05, 0.1, 0.5, 1, 2],
});

// Middleware
app.use((req, res, next) => {
  const start = Date.now();

  res.on('finish', () => {
    const duration = (Date.now() - start) / 1000;
    httpRequestCounter.labels(req.method, res.statusCode).inc();
    requestDuration.observe(duration);
  });

  next();
});

// Expor métricas
app.get('/metrics', (req, res) => {
  res.set('Content-Type', client.register.contentType);
  res.end(client.register.metrics());
});
