📝 Logs: Os Olhos da Aplicação

Registros estruturados do que realmente acontece no seu sistema

O que é um Log?

Um log é um registro de um evento que ocorreu em sua aplicação em um momento específico. Pode ser um erro crítico, um aviso, uma informação sobre o fluxo normal, ou até debug detalhado.

Logs são essenciais para:

  • 🔍 Investigar erros — "Por que falhou?"
  • ⚡ Entender performance — "Qual operação foi lenta?"
  • 🛡️ Detectar problemas de segurança — "Houve acesso não autorizado?"
  • 📊 Auditar operações — "Quem fez o quê e quando?"

Logs Não-Estruturados (❌ Evite)

Logs em texto livre são difíceis de parsear, buscar e analisar automaticamente: 

2026-05-03 14:23:45 ERROR Failed to connect to database
2026-05-03 14:23:46 INFO Retrying connection attempt 2
2026-05-03 14:23:48 ERROR Connection timeout after 5000ms

❌ Problemas:

  • Impossível buscar programaticamente
  • Sem contexto estruturado (userId, requestId, etc)
  • Difícil correlacionar com traces e métricas
  • Análise manual e propensa a erros

Logs Estruturados (✅ Recomendado)

Logs estruturados (geralmente em JSON) contêm campos bem definidos que podem ser processados por máquinas: 

{
  "timestamp": "2026-05-03T14:23:45.123Z",
  "level": "error",
  "service": "api-gateway",
  "message": "Failed to connect to database",
  "error": {
    "type": "ConnectionTimeout",
    "code": "ECONNREFUSED"
  },
  "context": {
    "userId": "usr_12345",
    "requestId": "req_67890",
    "environment": "production",
    "version": "1.2.3"
  },
  "duration_ms": 5000,
  "stack_trace": "at connectDB (database.js:42)"
}

✅ Benefícios:

  • Estrutura consistente — fácil de parsear
  • Contexto completo — correlação com requests e users
  • Queryável — buscar por `level=error` ou `userId=usr_12345`
  • Agregação — Loki, ELK, Splunk entendem automaticamente
  • Correlação — linkado com traces via `traceId`

Melhores Práticas

1️⃣ Use Logs Estruturados

JSON é o padrão de facto para logs em produção.

logger.info({ userId, requestId, action }, "User logged in")

2️⃣ Inclua Contexto Suficiente

Sem context, logs são inúteis. Sempre inclua:

  • requestId / traceId
  • userId ou sessionId
  • service (qual aplicação?)
  • environment (dev, staging, prod?)

3️⃣ Use Níveis Apropriados

  • ERROR — Algo falhou, precisa ação
  • WARN — Algo estranho, monitorar
  • INFO — Evento normal (login, compra)
  • DEBUG — Detalhes para troubleshooting

Em produção: INFO ou WARN. DEBUG apenas em staging/dev.

4️⃣ Agregue Logs Centralizados

Não leia logs de cada servidor separadamente. Use:

  • Loki — Leve, integrada com Grafana
  • ELK Stack — Elasticsearch, Logstash, Kibana
  • Splunk — Profissional, pago
  • Datadog — Cloud, com APM integrado

5️⃣ Correlacione com Traces

Use o mesmo traceId em logs, métricas e traces:

logger.info({ traceId: req.id }, msg)

Depois você consegue puxar TODOS os logs de um request específico.

6️⃣ Cuidado com PII

Nunca logue senhas, tokens ou dados pessoais. Use:

password: "[REDACTED]", creditCard: "****1234"

Exemplo Prático: Node.js com Pino

Aqui está como implementar logs estruturados:

import pino from 'pino';

const logger = pino({
  level: process.env.LOG_LEVEL || 'info',
  base: {
    service: 'api-gateway',
    environment: process.env.NODE_ENV,
    version: '1.0.0'
  }
});

// Uso
app.post('/login', (req, res) => {
  logger.info(
    { userId: user.id, email: user.email, method: 'password' },
    'User login attempt'
  );

  if (!user.passwordMatch) {
    logger.warn(
      { userId: user.id, ip: req.ip },
      'Login failed: invalid password'
    );
  }

  logger.info(
    { userId: user.id, sessionId: session.id },
    'User logged in successfully'
  );
});

Resultado em Loki:

{
  "timestamp": "...",
  "level": 30,
  "service": "api-gateway",
  "environment": "production",
  "userId": "usr_123",
  "sessionId": "sess_456",
  "msg": "User logged in successfully"
}

Ferramentas para Logs

Pino (Node.js)

Logger rápido e minimalista para Node.js

npm install pino

Winston (Node.js)

Logger flexível e extensível

npm install winston

Loki

Agregação de logs leve, integrada com Grafana

docker run grafana/loki

ELK Stack

Elasticsearch + Logstash + Kibana (pesado, poderoso)

docker-compose up -d elasticsearch
